什么是Web服务器？Web服务器软件有哪些？

什么是Web服务器？

我们最常说的Web服务器指的是网站服务器，它是建立在Internet之上并且驻留在某种计算机上的程序。Web服务器可以向Web客户端(如浏览器)提供文档或其他服务，只要是遵循HTTP协议而设计的网络应用程序都可以是Web客户端。

Web服务器的工作原理可以分为以下四个步骤:

①连接过程:是Web服务器与其浏览器之间建立的连接。检查连接过程是否实现。用户可以找到并打开虚拟文件套接字。该文件的建立意味着连接过程已经成功建立。

②请求过程:Web浏览器利用socket文件向其服务器发出各种请求。

③响应过程:在请求过程中发出的请求通过使用HTTP协议传输到Web服务器，然后执行任务处理。然后，通过使用HTTP协议将任务处理的结果传送到网络浏览器，并且在网络浏览器上显示所请求的界面。

④关闭连接:是最后一步——响应过程完成后，Web服务器与其浏览器断开连接的过程。Web服务器的上述四个步骤联系紧密，逻辑严密，可以支持多进程、多线程以及多进程、多线程混合的技术。

Web服务器软件有哪些？

目前主流的web服务器主要是Apache、Nginx、IIS，还有较多使用的Tomcat、Jetty、WebSphere，WebLogic，Kerstrel等。下图为市场占有率历史数据，Apache占有率较高，但是在前1K网站排名中，Nginx占有率最高。

什么是Web浏览器？Web浏览器软件有哪些？

什么是Web浏览器？

网页浏览器（英语：web browser），常被简称为浏览器，是一种用于检索并展示万维网信息资源的应用程序。这些信息资源可为网页、图片、影音或其他内容，它们由统一资源标志符标志。信息资源中的超链接可使用户方便地浏览相关信息。

Web浏览器软件有哪些？

主流网页浏览器主要指的是Mozilla Firefox、Internet Explorer、Microsoft Edge、Google Chrome、Opera及Safari。

安装一个网站软件与安装一个桌面软件有什么不同？

一、网站软件无需安装即可使用，桌面软件需要安装到本地。

二、一般来说桌面app相应速度更快。

三、桌面软件不需要浏览器支持，web应用程序依赖浏览器执行。

四、桌面app以Windows窗口形式存在。

五、桌面软件更新需要重新安装下载，web应用只要作者在服务器上安装以此就可以通过网络给所有
客户端共享，不需要重新下载.只需要修改服务器上Web应用程序，则所有的客户端自动进行更新。

安装hexo必须依赖哪些软件？这些软件各自的职能是什么？

  • Node.js:一个基于Chrome V8 引擎的 JavaScript 运行环境。它让JavaScript可以开发后端程序。

  • npm：node.js的包管理工具。安装hexo时用于下载hexo、hexo sever包。

  • Git：一个版本控制系统。在 hexo中主要用于执行清除缓存文件、生成静态网页、启动本地服务器和部署到云端服务器等操作。

运行hexo所生成的网站依赖哪些软件？

hexo生成的是静态网页

通常静态网页的工作机制为：

  1. 使用HTML编写静态页面，发布到WEB服务器端；

  2. 客户端通过浏览器请求该静态页面；

  3. Web服务器根据请求定位该静态页面；

  4. 该静态页面以HTML流的形式返回客户端；

  5. 客户端浏览器解释HTML流，并显示为WEB页面。

使用hexo生成一个网站，需要经历哪几个关键步骤？

首先需要安装依赖的软件

1. 安装hexo 任意路径 git bash here

npm install -g hexo-cli

2. 初始化站点、安装项目依赖包（第三方nodejs类库或工具）

命令说明：

hexo: 是hexo的命令

init: hexo的子命令，用于初始化一个博客系统．

blog: 是博客系统的名字，这里我没有考虑太多，直接将博客系统的名字命名为blog,　你可以根据自己的喜好设定名字．

初始化完成后的简要目录结构如下:

目录结构说明

package.json: 这个是nodejs要用到的，里面存放着项目信息，版本，项目所依赖的第三方nodejs工具和代码库

package-lock.json: 这个nodejs解析依赖包时会用到的

scaffolds: 博客脚手架，当你创建新的博文时可以基于其中的某个模板进行修改．

source: 源代码文件夹，这里存放你网站的内容．

themes: 主题，博客css样式，背景图片等等．

node_modules: 这里存放npm install时从网络下载下来的依赖包

config.yml 项目配置文件

配置文件详细说明

这里重点说明一下项目配置文件．

你可以修改项目的默认配置文件_config.yml以满足你个人的需求．比如标题，网站的描述，搜索关键字等等一些信息

3.启动博客系统

　hexo server

如果启动成功，会输出以下信息，最后一行会提示，通过什么网址可以访问博客．

打开浏览器，将网址粘贴到浏览器即可访问到博客系统．

4.创建文章

进入blog文件目录 git bash here

hexo new 文章名

然后进入source 目录 _post 目录下找到文章使用markdown语法编写文章。

使用wordpress构建的博客网站与使用hexo构建的博客网站有什么不同？

  1. WordPress 拥有完整的后台管理系统及数据库，编辑文章、管理界面等操作在后台管理系统完成。hexo 添加文章、魔改等操作都在本地文件操作，完成后生成静态页面，再部署到网络上。
  2. WordPress有丰富的主题和插件库。hexo主要用npm 包安装工具下载主题和插件。
  3. WordPress用古腾堡编辑器 在创作时将文章内容分为一个一个的“块”。 hexo使用 markdown 语法写文章，编辑文章后需要在本地生成静态网页，然后同步到服务器。
  4. 要部署到网上 WordPress要购买服务器，hexo有免费托管软件。
